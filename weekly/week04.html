Notes for Week 04

Things to Remember

  • Use Reset in forms sparingly
  • Default input is text, If I don't specify it will default to this. (see examples)
  • Always use literals to create an array
  • Can add new properties by assignment to classes
  • hasOwnProperty()
  • Own properties override the prototype
  • Never use arrays or objects as a default value in prototype
  • Setters and Getters
  • propertyIsEnumerable()

Research Needed

  • Form Design. see this book: Designing UX: Forms
  • Form input types: see this documentation: MDN Web Docs: Input Element
  • How OOP works in Javascript
  • Classes VS Objects
