Notes for week 2

Defining my Code conventions

  • Functions: use arrow syntax for one line 
                x = ()=>
    Everything else use standard: 
              myFunction(){}
  • Naming: camelCase
  • Variable declarations at beginning of blocks using let and const
  • Script Placement: below footer
  • Quotes: Use single quotes as default, template literals for using variables
  • Numbers: Assign numbers to variables before using dot notation
  • Conversions: use explicit conversion methods
  • Comparisons: Use hard equality and Number.isNaN()

Things to Remember

  • All functions return, undefined if not explicitly stated
  • Always consider hoisting and scope when declaring variables

  • Default parameters
